 1) В данном файле опишем создание дампа БД example и его разворачивание в БД sample
 Примечание: работа выполняется в ОС Windows, файл my.cnf создан и настроен с указанием [client]
 
 В командной строке выполним команду:
	mysqldump example > example.sql
	
 Войдем в mysql и создадим пустую БД sample:
	mysql
	CREATE DATABASE sample;
	exit
	
 Развернем дамп из файла example.sql в БД sample:
	mysql sqmple < example.sql

 2) В данном разделе опишем создание дампа БД mysql только с первой сотней значений таблицы help_keyword
 В командной строке выполним команду:
	mysqldump --where="true limit 100" mysql help_keyword > mysqldump.sql
	где:
	параметр --where="true limit 100" - задает условие дампа записей. В данном случае их количество не должно превышать 100;
	mysql - имя БД, дамп которой выполняется;
	help_keyword - имя таблицы, для дампа.
	
 Однако при попытке развернуть данный дамп в другую БД происходит ошибка с текстом:
    The table 'help_keyword' may not be created in the reserved tablespace 'mysql'
	
 Причины этого не совсем понял. Поиски в интернете намекают на то что это свяязано с использованием именно системной БД mysql.
 Если будет возможность - интересно было бы узнать реальную причину и возможно ли это исправить.
